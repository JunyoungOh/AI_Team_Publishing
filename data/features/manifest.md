# Enterprise HQ — 기능 레퍼런스

> 이 문서는 **가이드 챗봇의 유일한 지식 베이스**입니다. 각 사이드바 모드의 **기능**, **작동 방식**, **할 수 있는 일의 예시**만 담고 있어요. "언제 이걸 쓰세요" 같은 추천 맥락은 이 문서에 **미리 적지 않습니다** — 사용자의 실제 과제를 읽고 AI가 이 문서를 근거로 직접 판단해야 합니다.
>
> **수정 가이드**: 기능이 바뀌거나 새 모드가 생기면 이 파일을 편집하세요. 서버 재시작 없이 즉시 반영됩니다. 각 모드는 `## [slug] 모드 이름` 헤딩으로 시작해야 하며, slug는 사이드바의 `data-card-mode` 값과 일치해야 합니다 (`scripts/check_manifest_drift.sh`가 검사).

## [builder] 플레이북

**기능**
- **방법론을 먼저 설계해두고, 그 방법론으로 다양한 작업을 반복 실행**하는 모드입니다. "어떤 관점에서, 어떤 데이터를, 어떤 도구로, 어떤 형식으로 뽑아낼지"를 사전에 짜두고, 나중에는 작업 내용만 던져 바로 실행합니다.
- 한 번 설계된 플레이북은 리서치·기획·아이디어·파일 분석 등 다양한 작업에 재사용됩니다. 같은 플레이북이 매주 다른 주제를 같은 품질로 돌려줍니다.
- 첨부파일(엑셀·CSV·PDF·문서 등)을 input 폴더에 넣어두면 플레이북 실행 시 파일 기반 분석에 자동으로 활용됩니다.
- 저장된 플레이북은 자동실행 탭에서도 호출해 시간 기반 자동 실행이 가능합니다.

**작동 방식 (3단계)**
- **1단계 — 사용자 지시 명확화**: AI가 사용자에게 몇 가지 명확화 질문(목적·중점 관점·깊이·결과 형식 등)을 던져 만들고 싶은 플레이북의 윤곽을 정확히 파악합니다.
- **2단계 — 방법 설계**: 명확화 답변을 기반으로 AI가 분석 관점, 수집 데이터, 출력 형식을 정리해 strategy 카드(플레이북)로 저장합니다. 이 단계까지 끝나면 한 번 설계된 플레이북이 사이드바에 자산으로 쌓입니다.
- **3단계 — AI 실행**: 저장된 플레이북을 선택한 뒤 작업 내용(예: 어떤 회사를 분석할지)을 자연어로 던지면, AI가 그 플레이북 설계대로 자율적으로 리서치·분석·정리를 수행하고 결과 보고서를 생성합니다.
- 첨부파일은 사이드바 WorkspacePanel에서 선택하면 AI가 절대경로로 직접 열어 분석에 반영합니다.

**할 수 있는 일의 예시**
- "경쟁사 5곳을 매번 같은 5개 관점(시장점유율·재무·기술·인력·전략)으로 비교 분석하는 플레이북을 만들어줘"
- "이 엑셀 데이터를 받아서 매번 동일한 차트와 인사이트를 뽑아주는 플레이북"
- "내가 쓴 기획서 PDF를 던지면 사용자 페르소나 3개와 핵심 가치제안을 뽑아주는 플레이북"
- "주간 업계 뉴스를 같은 카테고리·같은 톤으로 정리하는 플레이북"
- "신제품 아이디어를 던지면 시장성·기술타당성·차별화 3관점으로 평가하는 플레이북"

## [schedule] 자동실행

**기능**
- 저장된 **플레이북을 정해진 시간에 자동으로 실행**합니다. cron 기반 시간 트리거.
- **저장된 어떤 플레이북이든** 자유롭게 골라 자동실행에 투입할 수 있습니다 (예전처럼 "스케줄용 타입"을 따로 만들 필요가 없어짐).
- 실행 대상은 (저장된 플레이북 + 작업 내용) 또는 (자연어 과제 단독) 둘 다 가능합니다.

**작동 방식**
- **이 앱이 실제로 실행되어 있어야만 자동실행이 돌아갑니다.** 앱을 끄면 예약된 작업도 멈춥니다 (서버가 꺼진 동안의 예약 시간은 건너뛰어짐).
- 등록된 자동실행은 내부 스케줄러에 저장돼, 앱이 켜져 있는 동안 지정된 시각에 자동 실행됩니다.
- 실행 이력과 결과 보고서가 누적 보관됩니다 (Delta 비교/Append 모드 지원).
- 이벤트 기반 트리거는 지원하지 않습니다 — 시간 기반 cron 전용입니다.

**할 수 있는 일의 예시**
- "매일 오전 8시에 '주간 업계 뉴스 요약' 플레이북 자동 실행해줘"
- "매주 금요일 17시에 '경쟁사 IR 리뷰' 플레이북으로 이번주 보고서 만들어줘"
- "매월 1일 '지표 리포트' 플레이북으로 지난달 결과를 자동 생성"
- "플레이북 없이도 매일 오전 9시에 'AI 업계 핵심 뉴스 5개' 자동 정리"

## [upgrade] 자동개발 (최초개발 + 강화소)

**기능**
- **개발 용어나 프로그래밍 지식 없이** "이런 게 있으면 좋겠다"는 생각만으로 AI가 실제 작동하는 앱을 만들어주는 모드입니다.
- React, Python, HTML 같은 기술 용어를 몰라도 됩니다 — AI가 알아서 고릅니다.
- 탭 안에 두 개의 서브모드가 있습니다:
  - **최초개발** — 0에서 1로: 머릿속에 있는 아이디어로 새 앱을 처음부터 만듭니다.
  - **강화소** — 이미 만든 앱에 기능을 추가하거나 아쉬웠던 부분을 고칩니다.

**작동 방식**
- **자동개발은 "내 컴퓨터에서 실행되는 로컬 앱"을 만들어줍니다.** 인터넷에 올리는 웹사이트나 온라인 서비스가 아니라, 내 컴퓨터에서 바로 켜서 쓰는 앱이에요.
- **최초개발**: 원하는 걸 **일상 언어로** 설명하면("○○해주는 앱이 있으면 좋겠어" 수준으로 OK), AI가 몇 가지 확인 질문을 던진 뒤 컴퓨터에서 바로 실행할 수 있는 앱을 자동으로 만들어줍니다.
- **강화소**: 이미 있는 앱 폴더를 알려주고 "여기에 ○○ 기능 추가해줘" 또는 "○○ 부분이 불편해, 고쳐줘"라고 말하면, AI가 먼저 앱을 파악한 뒤 필요한 질문을 되묻고 자동으로 개선합니다. 원본은 백업이 남으니 실패해도 원래대로 돌아올 수 있습니다.
- 개발이 진행되는 동안 AI가 뭘 하고 있는지 실시간으로 화면에 보입니다.
- 결과물은 실행 가능한 앱 파일들과 변경 이력, 그리고 (강화소의 경우) 원본 백업입니다.

**할 수 있는 일의 예시**
- (최초개발) "매일 회의록 파일을 넣으면 핵심만 요약해주는 작은 도구 만들어줘"
- (최초개발) "강아지 산책 기록을 사진과 함께 남길 수 있는 앱 만들어줘"
- (최초개발) "내가 읽은 책 목록을 예쁘게 정리해서 보여주는 앱 만들어줘"
- (최초개발) "우리 가족 식단표를 달력처럼 관리할 수 있는 도구 만들어줘"
- (강화소) "내가 만든 할 일 앱에 '오늘 뭐부터 할지 추천해주는' 기능을 넣어줘"
- (강화소) "이 메모 앱 글자가 너무 작아서 읽기 불편해, 전체적으로 보기 편하게 바꿔줘"
- (강화소) "내 메모 앱에 어두운 모드(다크 모드)를 추가해줘"

## [skill] 스킬

**기능**
- 자주 쓰는 **작은 서브태스크를 이름 붙여 저장**하고, 다른 모드에서 도구처럼 호출할 수 있도록 합니다.
- 스킬은 자연어로 정의한 작업 지시서 + (선택적으로) 필요한 외부 도구 목록으로 구성됩니다.

**작동 방식**
- 스킬 탭에서 대화형으로 스킬을 생성합니다 → `data/skills/installed/` 아래에 스킬 폴더로 저장되고 `data/skills/registry.json`에 등록됩니다.
- 등록된 스킬은 플레이북·자동개발 등 다른 모드에서 작업을 수행하는 AI가 도구처럼 호출해 사용할 수 있습니다.
- 스킬은 자동 트리거되지 않으며 명시적으로 호출될 때만 실행됩니다.

**할 수 있는 일의 예시**
- "입력 텍스트의 한국어 글자수를 세는 스킬을 만들어줘"
- "한국어 문단을 영어로 요약하는 스킬을 등록해줘"
- "URL을 받아 Open Graph 메타데이터를 뽑는 스킬을 만들어줘"
- "긴 회의록에서 결정사항과 액션 아이템을 분리해주는 스킬을 만들어줘"

## [discussion] AI 토론

**기능**
- 여러 AI 에이전트가 **서로 다른 입장**에서 하나의 주제를 토론하고, 사회자 에이전트가 합의·반대 의견을 정리합니다.
- 참여자(페르소나)는 페르소나 워크숍에서 만든 인격을 지정할 수 있습니다.
- **참여자는 최대 4명까지** 설정할 수 있습니다.

**작동 방식**
- 토론 주제 + 참여자 구성(최대 4명)을 입력하면, 각 참여자가 순차적으로 발언하면서 토론이 진행됩니다.
- 사회자가 타이머·합의 투표·정리를 주도합니다.
- 결과는 토론 로그(누가 언제 뭐라 했는지)와 최종 합의/반대 요약으로 저장됩니다.

**할 수 있는 일의 예시**
- "신제품 A를 올해 안에 출시해야 할지 찬성 2명 반대 2명으로 토론해줘 (총 4명)"
- "전기차 보조금 축소 정책의 영향에 대해 경제학자·환경운동가·자동차업계 3명 관점으로 토론"
- "새 채용 프로세스 초안에 대해 HR·법무·현업팀장·신입사원 4명 관점으로 리스크를 논의해줘"

## [foresight] 포사이트 (민들레)

**기능**
- 중장기 **미래 시나리오·트렌드**를 다중 에이전트가 상상·탐색합니다.
- 확정된 답이 아닌 **가능성의 발산**을 목적으로 하는 모드입니다.

**작동 방식**
- 탐색 주제 + 시간 지평(예: 5년, 10년)을 입력하면, 여러 에이전트가 서로 다른 궤적을 상상합니다.
- 각 시나리오의 분기 지점, 전제 조건, 파급 효과가 함께 정리됩니다.
- 결과물은 시나리오 보고서 형태로 저장됩니다.

**할 수 있는 일의 예시**
- "2035년 한국 모빌리티 시장의 시나리오 3가지를 그려줘"
- "AI 규제가 급격히 강화되는 미래에서 우리 사업은 어떻게 변할까"
- "원격근무가 완전히 사라진다면 어떤 연쇄 효과가 벌어질까"
- "기후변화로 농업 지도가 재편되는 시나리오를 탐색해줘"

## [persona] 페르소나 워크숍

**기능**
- 인터뷰 형식으로 AI **인격(페르소나)을 설계·저장**하고, 다른 모드(비서·토론)에서 재사용할 수 있게 합니다.
- 페르소나는 배경·말투·전문성·가치관이 담긴 상세 인격 설정입니다.

**작동 방식**
- 워크숍이 질문을 던지면서 인격의 속성을 채워나갑니다.
- 저장된 페르소나는 `data/personas/` 아래에 보관되며, 비서의 인격으로 교체하거나 토론 참가자로 투입할 수 있습니다.
- 다른 사용자와 공유 가능한 페르소나와 개인 페르소나가 구분됩니다.

**할 수 있는 일의 예시**
- "깐깐한 법무팀장 페르소나를 만들어줘 — 리스크 회피 성향이 강함"
- "내 스타트업의 멘토로 쓸 VC 파트너 인격을 설계해줘"
- "고객 역할극용 'B2B SaaS 구매 담당자' 페르소나를 만들어줘"
- "창작용 캐릭터: 1980년대 뉴욕 재즈 클럽 주인 인격"

## [secretary] AI 비서

**기능**
- **지속 대화형 AI 비서**입니다. 일정 관리, 메모, 짧은 조사, 다른 모드로의 과제 인젝션을 자연어로 시킬 수 있습니다.
- 대화 이력이 세션 간 유지되며, 페르소나를 교체해 인격을 바꿀 수 있습니다.

**작동 방식**
- 두 가지 응답 모드: **Flash**(Haiku, 빠른 단발 응답)와 **Think**(Sonnet + extended thinking, 복잡한 판단용).
- 대화 중 "이건 플레이북으로 넘겨줘" 같은 명령으로 다른 모드에 과제를 인젝션할 수 있습니다.
- 일정은 `data/calendar/`, 대화 이력은 `data/secretary/`에 저장됩니다.

**할 수 있는 일의 예시**
- "내일 오후 3시에 '팀 회의' 일정 추가해줘"
- "이 회의록을 요약하고 할 일 3개만 뽑아줘"
- "오늘 해야 할 일이 뭐였지?"
- "이 아이디어를 플레이북으로 넘겨서 심화해줘"

## [law] AI 법령

**기능**
- **law.go.kr 공식 Open API**를 호출해 한국 법령의 원문 조문을 인용하며 답변합니다.
- 답변에는 근거 조문이 `[인용]` 블록으로 함께 붙습니다.
- 한국 법령 전용입니다 — 해외 법령, 판례, 해석론은 다루지 않습니다.
- 두 가지 방식의 검색을 모두 지원합니다:
  - **키워드 검색** — 특정 법령 이름·조문 번호·용어로 찾기 ("근로기준법 52조", "영업비밀")
  - **상황 검색** — 실제 상황을 설명하면 그에 맞는 법령을 찾아주기 ("야근 수당을 못 받았는데 어떤 법이 적용돼?")

**작동 방식**
- 사용자가 **구체 법령 이름·조문**을 알고 있으면 바로 그 조문을 찾아 인용합니다.
- **상황만 설명**하면 AI가 그 상황에 적용될 수 있는 법령을 먼저 판단한 뒤 관련 조문을 가져옵니다.
